# 2020-07-10

# input 相关

  • 如何控制 <input type="number" />不能输入负号
    $(document).on('input','.over_commission',function(e){
    e.target.value = e.target.value.replace(/\-/g,""); 
});
  • input 表单禁止粘贴: <input onpaste = "return false"/>
  • input 表单点击输入框全选旧内容: <input onclick="this.select()"/>
  • inpput 表单获取焦点时全选: <input onfocus="this.select()"/>

# 下拉选和单选按钮

  • 单选按钮和下拉选 获取选中的值 以及 根据后端返回的值渲染页面.
    // 单选
<html>
    <div>
        <span>年假:</span>
        <input type="radio" name="annual_leave" value="0">正常出勤
        <input type="radio" name="annual_leave" value="1">未出勤
    </div>
    <div>
        <span>级别:</span>
        <select id="level">
            <option value="1">一级</option>
            <option value="2">二级</option>
            <option value="3">三级</option>
            <option value="4">四级</option>
        </select>
    </div>
</html>
<script>
// 获取选中的值
    // 单选按钮
    $('input[name="annual_leave"]:checked').val();
    // 下拉选
    $('#approval_level option:selected').val();
//渲染页面
    // 单选按钮: 
    $(`input[name="annual_leave"][value="${list[i].is_normal}"]`).attr('checked',true); 
    // 下拉选: 
    $(`input[name="level"][value="${item}"]`).attr("checked",true);
</script>

    // Bug:PC 后台多次点击进入一个页面,让下拉选中的一项被选中,多次进入后就会出现下拉选多项被选中的情况,为了避免这种情况,需要在点击进去的时候,去除该下拉选中的所有选中项,然后再根据后端返回的值,进行选中.
$('#dishContractType option').each(function(){
    $(this).attr('selected',false);
});
$(`#dishContractType option[value='${data.data.dish_contract_type}']`).prop('selected',true);
// 问题: 通过 attr 进行选中赋值的时候,查看元素确实有属性“checked=checked”,但是checkbox却没显示选中:
// 解决: 从jQuery1.6开始新增了一个方法 prop(),因为 attr()有时候会出现不一致的行为,所以官方建议:具有 true 和 false 两个属性的属性,如 checked,selected,或者 disabled使用prop(),其他的时候 attr().
